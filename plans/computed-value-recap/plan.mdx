# ComputedValue: elimina boxing no avaliador

Branch `feature/computed-value` sobre `main`. Migração incremental para trocar o retorno `object?` do
avaliador por um value type `ComputedValue`, eliminando o boxing. Três capítulos: o **experimento** que prova
o ganho, a **Fase 1** (tipos core, aditivo) e a **Fase 2** (o *seam* `Evaluate` + coerção nativa + nós-valor).

**Motivação medida:** o cache do avaliador (`Dictionary<…, object?>`) boxa **24 B por célula** de vida longa
→ dispara coletas **Gen1**. Com `Dictionary<…, ComputedValue>`: **0 B, zero coletas**. No caminho aritmético
o struct é 2,3×–4,3× mais rápido; no caminho de extração em lote o ganho é 4–12% de throughput — mas o
argumento forte é memória/GC num extrator de background.

**Estratégia sempre-verde:** a base ganhou `Evaluate(ctx) : ComputedValue` com bridge default
`From(Compute())`. Cada nó fica em um de dois estados consistentes; migrar = mover do estado "legado" para o
"nativo", um nó por vez, com a suíte sempre verde (240/240). Nada breaking ainda — a virada da assinatura
pública de `Compute` fica para fases seguintes.
